# 🚀 Order Service (Idempotency + Redis + Performance Testing)

Сервис для создания заказов с поддержкой **идемпотентности**, **Redis**, **PostgreSQL**, **rate limiting** и нагрузочного тестирования через **k6**.

Проект демонстрирует решение реальных backend-проблем:

- race condition при конкурентных запросах
- дублирование данных
- устойчивость под нагрузкой
- защита от перегрузки

Репозиторий показывает, как QA/AQA инженер может не только находить баги, но и **анализировать, воспроизводить и устранять проблемы production-уровня**.

## Идемпотентность (Idempotency)

Сервис гарантирует:

- повторный запрос с тем же ключом НЕ создаёт новый заказ
- все клиенты получают одинаковый результат

### Реализация:

- Redis используется как быстрый storage для ключей идемпотентности
- атомарные операции предотвращают race condition
- Защита от перегрузки (Rate Limiting)
- ограничение количества запросов
- возврат 429 Too Many Requests
- защита сервиса от DDoS и перегрузки

## 📬 API

## Создание заказа


## Headers

```text
Idempotency-Key: <UUID>
```

## Ответы

| Код | Описание                           |
| --- | ---------------------------------- |
| 201 | заказ создан                       |
| 200 | повторный запрос (идемпотентность) |
| 409 | конфликт                           |
| 429 | превышен rate limit                |

## 🐞 Найденные проблемы

## 1. Race condition

Проблема
- При конкурентных запросах создавались дубликаты заказов.

Решение
- добавлена идемпотентность через Redis
- использованы атомарные операции

## 2. Ошибки под нагрузкой

Проблема
- Высокий процент failed requests.

Решение
- оптимизирована логика обработки
- снижена нагрузка на БД

## 3. Поведение при перегрузке

Решение
- добавлен rate limiting
- возврат корректного статуса 429
